Recommended actions

  1. Strengthen card-not-present monitoring, especially overnight.
  2. Combine amount deviation, category, merchant history and customer baseline.
  3. Require minimum volumes before escalating geographic/job segments.
  4. Repair location keys before production joins.

Data caveat

The separate location file has only 39.95% reliable key coverage because card numbers were stored in scientific notation. Coordinates in the transaction table power geographic analysis.

Metric definitions and source

Fraud rate = fraud-labelled transactions / all transactions in the segment. Fraud amount = sum of transaction amount where is_fraud = 1. Source: cc_data_dec19.csv, supplied project dataset. All customer identifiers and direct personal fields are excluded.
